# CanyonCruise

## Description 
I developed a web application that connects hikers, skiers, and trail runners to share rides simply and reliably. The platform allows users to offer or request rides based on location and time, helping outdoor enthusiasts reach trailheads and resorts more efficiently. In addition to improving accessibility, the app gamifies sustainability by awarding users achievements for reducing the number of vehicles on the road and lowering overall CO₂ emissions.

## Features
- User log-in and authenticaiton with email and password
- Ability to offer and request rides
- Ride matching based on location and time  
- In-app chat for communication between driver and rider
- Active bookings dashboard
- Browse all available rides
- Milstone tracking, including carbon savings count, rides offered, rides requested
## Tools
- Front-end: SvelteKit with TypeScript, Tailwind CSS
- Back-end: Supabase (authenicaiton, database), VSCodium
- Other: AI assisted developement using ChatGPT and Cursor Pro

## Relationship to BCIS program goals

This project was designed to apply and integrate key concepts from the Business Computer Information Systems (BCIS) program. A primary objective was to practice the full software development lifecycle, including planning, design, testing, and implementation. I achieved this by building CanyonCruise from the ground up, which required thoughtful consideration of both system architecture and user experience.

Additionally, the project reflects the application of business, statistical, and technical principles to solve a real-world problem. I identified a transportation inefficiency within the outdoor recreation industry and developed a platform to address it. The solution incorporates both technical functionality and business strategy, including potential partnerships with gear shops, outdoor organizations, and ski resorts to support adoption and growth
